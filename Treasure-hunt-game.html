Treasure hunt game



I started working on this project because while working with Greenfoot in the first course I took, after a few weeks I started asking myself how Greenfoot actually works under the hood and how the functions that do all the heavy lifting for you work.
I also wanted to get into "real" programmig, because Greenfoot after I got a little used to it, gave me the same feeling scratch gave me, which is that outside of the programming environment of Greenfoot I have no ideea how to program anything.
So I decided to try "real" programmig.
I downloaded vscode and the JVM, and as soon as I saw "public static void main(String[] args)" I thought to myself I know even less about Java programming than I knew before, so I decided to learn how to program what Greenfoot offers you from scratch in vscode. I searched for a tutorial on Youtube and found a really good course made by Ryisnow that offers exactly what I was looking for.
And so this project is what the course offered by Ryisnow helps you build in the first part of his series of 60 videos.

In the you spawn in the middle of an island and have to find the treasure, but you will encounter some difficulties, like locked doors and fallen branches, to get past these obstacles you have to find the appropiate tools.
there is also a timer so you can compete with friends and family to see who can get the best score!

I first had to learn how to create a window in which I could later draw my game into.

I then had to learn about the gameloop and the different implementations there are.

I learned a lot more from this project but another thing I learned and still find interesting is the world coordinate system and the screen coordinate system, I learned how to convert the world coordinates into screen coordinates and used that to create a camera effect that follows the player character.

Other things I learned:
  • Key handling
  • Even handling
  • Collision checking
  • Sound implementation
  • UI
