Ransomware



This was the first time I wrote a programm in Python except for when I first installed the Python intepreter and wrote a "Hello, World!" programm.
I got the inspiration for this project from a Youtube short which showcased how you can use the fernet module from the Python cryptography library to encrypt/decrypt a certain file.
I tried it myself, and then decided I want to bring it a step further and also use that as an opportunity to get more familiar with Python.
What I thought about adding was recursivity, so that is what I decided to add along with some features to make it more user friendly and "safe" in the sense that I implement some sort of failsafe or warning the user gets if for example the user tries to encrypt a file that has already been encrypted, since that would make it impossible to decrypt if the user didn't save the key of the previous encryption.

There are two programms, Encrypt.py and Decrypt.py.
Encrypt.py goes through all files and directories inside the directory in which the programm was executed and first checks if they have already been encrypted, if it finds a file that has already been encrypted it warns the user and asks if the program should continue execution.
It then lists all files that will be encrypted, if there are files that fail to encrypt, it also lists those files.
After the encryption it creates a key.txt file in which the key used to encrypt the files is stored.
Decrypt.py first reads the key.txt file, if it doesn't exist it notifies the user and stop execustion.
If key.txt was found it goes through all files and directories inside the directory in which the programm was executed and recursively decrypts all files.
It then prints all decrypted files, and all files that failed to decrypt or have been ecncrypted more than once (if there are any).

See project on Github

Encrypt.py

The program first generates a key then gets a list of all files in the current directory, and then 4 arrays get initialized, one for files that will get encrypted, one for files that got encrypted, another for files that failed to encrypt and the last one is for the errors that came along with the files that failed to encrypt.
There is also a variable UNIQUE_STRING which is a string of choice, which helps identify already encrypted files.

The program then checks if there are any files that have already been encrypted by checking if the current file contains the unique string at the beginning.
If it finds a file that has already been encrypted it tells the user and asks if they want to continue encryption.
All files get added to files_to_encrypt.

Then all files that will get encrypted are printed out for the user.

encrypt_recursive is called next, encrypt_recursive encrypts all files recursively and any files that fail to encrypt are added to failed_to_encrypt and their error is added in failed_to_encrypt_error.

log_encrypted prints out all the files that have been successfully encrypted.

log_failed_to_encrypt prints out all the files that have failed to encrypt along with the error that caused encryption to fail.

At last the key used to encrypt all the files is stored inside key.txt.

Decrypt.py

The program first checks if the key.txt file exist, if it doesn't it terminates execution, else it continues and opens key.txt, gets a list of all files in the current directory and initializes 3 arrays, one for decrypted files, one for files that failed to decrypt and one for the errors of the files that failed to decrypt.

log_decrypted prints out all decrypted files.

log_failed_decrypted prints out all the files that failed to decrypt.

At last the key.txt file is deleted.
